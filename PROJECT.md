### 1. 赛博化羊皮纸顶级视觉契约 (Visual Overawe Theme Patterns)
所有的前端样式、布局逻辑全权掌握在位于系统表层根下唯一的入口应用文件 `app.py` 中。

### 2. 服务器终极云端部署落地的苛刻选择陷阱 (The Fall of Vercel Serverless Function)
整个世界虽然大量开源云部署喜欢把各类静态与反应流交托给云上巨无霸平台之一 **Vercel** 负责收尾，但是对 Streamlit 在此之上的运行我们需有着极度最深切的规避和退路设置：不能碰。
- **致命的协议天堑鸿沟限制**：一旦陷入了极为底层被死死焊住管控极差的 Vercel Hobby 版本底层 Serverless 云端服务器函数生态体内，极度的短时间上限紧缩断代（最多提供微弱怜悯极其可怜仅仅最大高达区区 `10 秒钟` 就必须要断网掐没死绝），以及平台原生极其痛恨对于双工全链路交互的长久存续 WebSocket （这也是决定一切 Streamlit 控制信号能够传导的基础设施），将决定硬生生传把此项目抛上 Vercel 只会彻底死火，换回一张惨烈至极满眼 `504 Timeout Gateway HTTP` 或者假动作断绝长锁连接异常暴毙白墙面。
- 所幸，我们在现有的整个庞大工程大版本架构设计时已敏锐避坑。内置所留下的一条细微旁支和静态声明配置组（即 `vercel.json` 配合 `api/index.py`）只是一层被当做掩人耳目拦截降级并用来承接不小心误访问到 Vercel 链接或者作为临时 API 中继过渡跳板使用的简单静态退路宣告（Fallback Landing Statement / API Proxy），去通知不知情的看客请把关注与视线转移至部署最长挂稳定服务的渲染大机房（诸如 Streamlit Community App Cloud，或是更原始且底层的能够持续不间断供给超长时间长耗时复杂大机器逻辑推理计算运转以及永久守护常驻在机器内部内存的长驻大管道计算大容器机群渲染如 Render / Zeabur 上等等）。
