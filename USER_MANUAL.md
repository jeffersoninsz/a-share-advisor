## 🌟 系统简介

**A股 AI 量化分析顾问** 是一款专为 A 股市场打造的**个人私有化智能投研平台**。系统通过调用大规模语言模型（Claude 3.5 Sonnet / OpenRouter 等），结合来自 AKShare 和 Efinance 的实时财经数据，为您提供深度、前瞻、多维度的独家股票分析报告。

我们抛弃了枯燥的原始传统界面，为您精心雕琢呈现了**沉浸式的「黑金赛博 + 羊皮纸」神秘卷轴风 UI**，让您在阅览冰冷数字的同时，体验顶级私人投行顾问般的视觉奢华享受。

### ✨ 核心亮点

- 🤖 **全自动情报搜集**：单次分析自动调动 9 大金融数据接口（包括 K 线趋势、宏观指标、财联社快讯、各路资金流向等），全方位扫描目标资产。
- 📊 **黑金赛博视觉**：响应式设计的深度暗黑羊皮纸风格，完美适配超宽 PC 端与移动端小屏。
- 🧠 **透明的思维链推理**：拒绝黑盒建议！每一条买卖意见都附带置信度评析和极具逻辑的完整链式推导过程。
- 🎯 **自省式准确率回测**：内置胜率计算引擎，历史测算准确留痕，支持手动标记实际盈亏情况，长期追踪机器系统真实的测算胜率。

> **⚠️ 郑重免责声明**
> 本系统提供的所有分析报告、买卖推荐及数据解读，**仅供金融编程学习与量化研究参考，绝对不构成任何直接的投资、理财或交易建议。** 股市环境瞬息万变，请您独立进行风险评估。由于使用本系统导致的任何资金/财产损失，开发者概不负责。

## ⚙️ 环境准备与安装

为了确保系统稳定出色的运行，请您按照以下指引步骤搭建基础环境。

### 1. 基础系统要求
- **操作系统**: Windows 10/11 (推荐原生支持), macOS 或各类主流 Linux 发行版
- **运行环境**: Python 3.10 或更高版本 (不建议使用过低版本)
- **网络连接**: 必须能够通达全网连接（本系统设计大量金融接口采集和外网大语言模型 API 通信）

### 2. 获取代码与安装依赖
请打开您的各类终端（如 PowerShell 或 Terminal），按顺序执行：

```powershell
# 1. 导航到项目的绝对根目录
cd "c:\Users\83158\Desktop\quantitative analysis"

# 2. 激活系统的 Python 虚拟环境 (必要步骤)
.venv\Scripts\Activate.ps1

# 💡 提示：如果 Windows 提示 PowerShell 执行策略错误，请先以管理员身份执行该命令解除封锁：
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 3. 安装或强制更新所需系统依赖
pip install -r requirements.txt
```

## 🔑 核心配置指南

在本系统运行前，需要为其配置 "AI 大脑" 的密钥与身份。系统默认读取根目录的 `.env` 文件。

### 1. 配置 API 密钥与代理
如果项目根目录下没有 `.env` 文件，请复制 `.env.example` 并重命名为 `.env`。

```ini
# 【必填】在此处填入您的 Anthropic 或 OpenRouter API Key
ANTHROPIC_API_KEY=sk-or-v1-xxxx-your-key-here

# 【必填】如果您使用的是中转服务（如 OpenRouter），请必须填写对应的 Base URL。如用官方则留空。
ANTHROPIC_BASE_URL=https://openrouter.ai/api/v1

# 【配置】所调用的模型全名名称 (此名称必须与对应 API 提供商吻合)
CLAUDE_MODEL=anthropic/claude-3.5-sonnet

# 【配置】分析长篇报告单次生成的最大 Token 限制量
MAX_TOKENS=4096
```

### 2. 自定义您的「关注白名单」
打开 `config/settings.py` 文件，这里面定义了能够被您的专属 AI 首先提权关注并分析挖掘的底层股票池：

```python
WATCHLIST = {
    "600519": "贵州茅台",
    "300750": "宁德时代",
    "000001": "平安银行",
    # 按照 "股票代码": "公司名称" 的格式继续添加您感兴趣的任意 A 股...
}
```

### 全局服务启动
确保您目前已经成功激活虚拟环境 `.venv` ，然后键入：

```powershell
streamlit run app.py
```
终端将会回显并自动让浏览器为您拉开位于 `http://localhost:8501` / `8503` 的神秘顾问控制台。尽情享受！

这里是您日常停留最久的核心大盘和工作台。

## 🧠 AI 引擎分析机制

您也许好奇，如此宏大深度，结构清晰的一份数千余字的硬核专业分析报告是如何在几十秒级别内被推演合成完成？

该神枢大脑建立在极其超前的现代大语言模型 **Tool Use (原生函数自动触发) 多轮博弈流转机制**上做重度定开。并且我们**严拒依赖**类似 Langchain 这种拥肿而黑盒的二层框架：这能够保证全速的推导流！

在一整次分析的生命周期中，Agent 拥有跨网络随时可自由传唤的数据情报中心——**9 大高并发数据接口探针**：

| 情报源函数代号 | Agent 职能描述 | 原始底层生态链 |
| :--- | :--- | :--- |
| `get_stock_list` | 提取当前您的系统监控自白名单 | Config (配置文件) |
| `get_realtime_quotes` | 指派快速并发抓取所有目标池内的最新实时现价、市盈率与涨跌幅 | efinance (东财) |
| `get_kline_data` | 获取各目标标的关键点位之日级别/周级别历史完整连贯 K 线流 | efinance (东财) |
| `get_kline_summary` | 使 AI 对股票数据实现降维打击，执行智能均线聚合平滑计算 (提取 MA5, MA20 行情带) | Pandas |
| `get_news_feed` | 7x24 小时不间断秒级抓取财联社最新金融电报，掌握第一手资讯脉动 | AKShare 接口簇 |
| `get_macro_data` | 高维度嗅探国家级与核心重磅公开的最新 CPI, PMI, 及外汇宏观流变指标 | AKShare 接口簇 |
| `get_sector_flow` | 俯瞰和深度监控近三日内各个 A股板块间极端的整体资金大进大出动向 | AKShare 接口簇 |
| `get_north_flow` | 定向分析剥离北向资金 (被常称为聪明钱) 之异常仓位变动和资金博弈状况 | AKShare 接口簇 |
| `get_history_reports`| 高阶认知：让 AI 自行通过阅读它以往在同样位置自己曾经下过的判断和逻辑报告，形成极其罕见的闭环“长程思考纠偏记忆” | SQLite |

每一次请求，您的专属终端 Agent 会在一座安全与网络被隔绝保护的无污染沙盒里，结合上下文进行不高于 `15轮` 的极速信息调取交锋与多级推理验证。成功拉取并处理超过几十个 KB 各类枯燥生冷的数字流后，执行归纳整理为严谨的 JSON 结构最终渲染为极佳美感阅读页面给您呈现。

## ☁️ 服务器部署指南

您如果希望将其部署公开网络成为一款全天候监控终端并和合伙人进行共享，请不要试图依赖传统的便宜短视距服务流来规避它：

### ❌ (绝对避坑) 无法支持直接 Vercel Serverless
请谨记，在云原生环境中 Vercel 免费版 (Hobby版) 的 Serverless 云函数有着**极其苛刻且容易被杀死的 10秒 Max-Timeout 锁死限制**，并且原生上对于 WebSocket 长并发请求采取屏蔽抛弃状态。而所有的 Streamlit 系统与页面刷新反馈正是极度建立在 WebSocket 长链接基石上的。如果您强行构建并部署进去，得到的仅仅是一面由于底层接口全阻断而抛出连续 `504 Timeout Gateway HTTP` 或者假死空载的白屏死区。本代码仓库内置的 `vercel.json` 和 `index.py` 退路是为了能够抛出降级静态宣传回落，并**无法用此进行时间长达数百秒的复杂 AI 大盘分析引擎**的实际运算！

### ✅ 最优完美白嫖级全链部署方案 (极度推荐 Streamlit Community Cloud)
这无疑是最原汁原味的、也是官方专门应对其生态量身做出的最优部署。而且不仅顺滑且近乎零成本！
1. 直接在您的 Github 里克隆并私有（公开也会自动适配）推送此全套源代码组合，务实确保 `requirements.txt` 和根目录处于绝对完好。
2. 转站跳转并绑定登录 [share.streamlit.io](https://share.streamlit.io/) 平台。
3. 右上角创建 New App (点击拉取您的这个代码仓库地址)，并且将入口精确制定给最重要的基底： `app.py` 启动锚点。
4. **【极为关键】** 在完成创建点击部署（Deploy）之前，在侧边设置进入 `Advanced Settings`（高级设定）菜单栏。找到环境变量挂载区。把我们曾经藏在本地 `.env` 中的核心密码组（含 `ANTHROPIC_API_KEY`, `ANTHROPIC_BASE_URL`）统统粘在系统内！确认保存，静待它的自动拉挂即完美跑起！

## ❓ 常见问题排查精编指南 (FAQ)

### 🔴 Q1：一点击「运行分析」，满江报错跳出来 `ModuleNotFoundError: No module named 'openai'` 等
**A:** 最普遍的通病！原因在于您没有按照首要规则激活本地专属于本生态沙盘的“隐秘隔离防护罩”（Virtual Environment 虚拟环境），又或者您的虚拟保护壳环境的底层依赖被污染断层过了。
> 解决药方：关闭一切占用窗口并把终端终止，重新输入 `.venv\Scripts\Activate.ps1` 唤醒环境前缀。接着怒敲 `pip install -r requirements.txt`，把所有必须包统统强制复权升级一遍然后再次 `streamlit run app.py` !

### 🟠 Q2：程序刚连按下去就闪退中止提示 HTTP `400 Bad Request: Invalid Model` 或无限卡死大盘分析
**A:** 这是一类直指 API 通信的灾害性故障提示！大概率因为您的 `.env` 内部那个称作 `CLAUDE_MODEL` 名字配置与官方目前提供或者中转接口提供商正在启用的标准参数字串出现了分歧。很多 OpenRouter 或者三方提供商需要的不仅仅叫 claude，它甚至精确为类似 `anthropic/claude-3.5-sonnet:beta` 或者其他附加限定名的魔化长串。
> 解决药方：强烈去审查并比对并严格改写您的环境文件内的变量名字段与其供应方出具的最底层技术要求保持完全丝毫不二致。

### 🟡 Q3：感觉页面黑了但是左上角的控制隐藏浮动栏位（侧边栏召唤按钮）神秘消失了？
**A:** 前端的代码中，本版本的暗金赛博全案 CSS 代码已经经过工程师对 DOM 最繁重与细密的彻底重写排版设计。任何原先前几版本曾经用来覆盖 Streamlit 最底层头部所导致的恶性后果 `header {visibility: hidden;}` 的滥杀隐患都已经被强力拔除净化过了，所有的侧边按钮功能已经获得最完美绝对性的长驻支持。如果您还是诡异地看不到任何图标？
> 解决药方：浏览器强行霸占了过时的主缓存。立刻使用 `Ctrl+F5` 或 Mac 系统按住 `Command+Shift+R` 暴力震荡并重载深度缓存一次即可！

### 🟢 Q4：切换到了我的随身小屏幕设备（手机）上翻看，大量密密麻麻的个股文字甚至数据全挤歪或者重合成了乱码块怎么破？
> 解决药方：如果您正遇到某款浏览器极为小屏异类的切割，您仅仅需要将屏幕**稍微横屏旋转换个方向持握**，极为霸气大气的超广排版马上就会华丽尽收眼底。

### 🔵 Q5：我在前端页面（或本地的 .env）修改了 API Key，但是跑分析依然报错，或者提示用的还是旧的？
**A:** 这是因为最早期系统的 Python 架构机制导致，在刚启动时 `agent/brain.py` 就把密码提前加载锁死在了内存中。之后不管你怎么从网页修改了文件，底层的核依然拿着上个世纪的旧密码去叩门！
> 解决药方：目前的引擎（V1.1）已由工程师彻底重组并打入了**热更新探针**。现在的无论本地通过页面改，还是直接改 `.env`，都会被下一次跑分析时立刻热捕获！
> **⚠️ 但极度注意 (Streamlit Cloud 玩家必看)**：如果您是把本系统托运上云（Streamlit Community Cloud），因为云端极其特殊的**“每次休眠唤醒即刻重制抹除所有网页操作和文件变动”**机制，您在云端网页无论填多少遍都没有用！必须登录 Streamlit 控制台后台 -> 进入该 App 的 `Settings` -> 左侧点 `Secrets` 填入配置才可实现永驻！

### 🟣 Q6：把我这套神级大盘部署上 Streamlit Cloud 后，一跑分析居然抛出血红报错 `PermissionError: [Errno 13] Permission denied: '/home/admin/.efinance'` 或诸如此类？
**A:** 这属于一种极为罕见的**“三方底层依赖库云端水土不服症”**。由于大盘极其依赖的底层核心探针库 `efinance` 有一个极蠢的习惯，它一旦启动，就会放肆地前往操作系统的极高权限只读区（比如主目录的 `/.efinance`）试图强行建立一个长期缓存数据屋。而极其注重安全的云平台当场就会把这种非法越权建房动作掐死，抛出无权限异常崩溃！
> 解决药方 (已根治)：我已经提笔在主程序的最高入口 `app.py` 的顶端植入了一段**神级拦截补丁 (Monkey Patch)**。系统现在开机点火瞬间，会立刻前驱劫持掉所有建立文件的底层指令，一旦扫描到是被拒绝的越权行为，就会非常优雅的把缓存文件夹请求强制引流重定向到我们自带的 `storage/efinance_data` 安全合规容灾屋里去。您只需要安心把最新代码推送上云端，这一顽疾即刻自动灰飞烟灭！
